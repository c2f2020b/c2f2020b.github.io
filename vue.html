v-bind #

讓我們看看下面的範例程式，在 Template 中的 v-bind:class="titleClass" 會找到 data 中的 titleClass ，並取出他的值 titleRed 作為 h1 標籤裡 class 的值。

vue
<script>
export default {
  data() {
    return {
      titleClass: 'titleRed'
    }
  }
}
</script>

<template>
  <h1 v-bind:class="titleClass">把我變藍色</h1> <!-- add dynamic class binding here -->
</template>

<style>
.titleRed {
  color: red;
}
.titleBlue {
  color: blue;
}
</style>

轉換後 HTML 的樣子 #

html
<h1 class="titleRed">把我變藍色</h1>

v-on #

v-on 主要是處理事件監聽，寫法有以下兩種，一般寫法及縮寫結果都是一樣的。

一般寫法

vue
<button v-on:click="increment">{{ count }}</button>

縮寫寫法

vue
<button @click="increment">{{ count }}</button>

下面是一個計數器，會計算我們點了幾次，點擊按鈕會呼叫 methods 中的 increment() ，在 methods 裡面，我們可以用 this 存取元件的實體，修改 count 的值，然後顯示在畫面上。

vue
<script>
export default {
  data() {
    return {
      count: 0
    }
  },
  methods: {
    increment() {
      this.count++
    }
  }
}
</script>

<template>
  <!-- make this button work -->
  <button v-on:click="increment">我點了{{ count }}次</button>
</template>

v-model #

v-model 具有雙向綁定的功能，下面的範例在 input 中輸入文字時， p 裡面的文字也會同步更新。

vue
<script>
export default {
  data() {
    return {
      text: ""
    };
  },
  methods: {}
};
</script>

<template>
  <div>
    <input v-model="text" placeholder="請輸入字串" />
    <p>{{ text }}</p>
  </div>
</template>

v-if 和 v-else #

v-if 使用上非常方便，下面是一個範例，點擊按鈕時，會自動切換好吃或是不好吃，並顯示正確的 h1 標籤內容。

vue
<script>
export default {
  data() {
    return {
      tastegood: true
    }
  },
  methods: {
    toggle() {
      this.tastegood = !this.tastegood;
    }
  }
}
</script>

<template>
  <div>
    <button @click="toggle">好吃?不好吃?</button>
    <h1 v-if="tastegood">好吃! 😀</h1>
    <h1 v-else>不好吃 😢</h1>
  </div>
</template>

v-for #

使用 v-for 可以依序渲染出整個陣列內容，下面是一個簡單的 Todo 應用程式。

vue
<script>
// 給每個 todo 一個唯一的 id
let id = 0

export default {
  data() {
    return {
      newTodo: '',
      todos: [
        { id: id++, text: '學習 HTML' },
        { id: id++, text: '學習 CSS' },
        { id: id++, text: '學習 JavaScript' },
        { id: id++, text: '學習 Vue' }
      ]
    }
  },
  methods: {
    addTodo() {
      this.todos.push({ id: id++, text: this.newTodo})
      this.newTodo = ''
    },
    removeTodo(todo) {
      this.todos = this.todos.filter((t) => t !== todo)
    }
  }
}
</script>

<template>
  <div> 
    <form @submit.prevent="addTodo">
      <input v-model="newTodo">
      <button>新增 Todo 項目</button>    
    </form>
    <ul>
      <li v-for="todo in todos" :key="todo.id">
        {{ todo.text }}
        <button @click="removeTodo(todo)">刪除</button>
      </li>
    </ul>
  </div>
</template>

watch #

原本網頁載入後，點擊按鈕下方的json 就不會再更新第二次了，不過藉由 watch 監控 todoId的變化來觸發 fetchData()，畫面下方的資料就可以同步更新。

vue
<script>
export default {
  data() {
    return {
      todoId: 1,
      todoData: null
    }
  },
  methods: {
    async fetchData() {
      this.todoData = null
      const res = await fetch(
        `https://jsonplaceholder.typicode.com/todos/${this.todoId}`
      )
      this.todoData = await res.json()
    }
  },
  mounted() {
    this.fetchData()
  },
  watch: {
    todoId(){
      this.fetchData()
    }
  }
}
</script>

<template>
  <p>Todo id: {{ todoId }}</p>
  <button @click="todoId++">Fetch next todo</button>
  <p v-if="!todoData">Loading...</p>
  <pre v-else>{{ todoData }}</pre>
</template>

Components #

父元件引入子元件範例，在父元件的 script 區塊中 import 子元件，並在 export default 裡面加上子元件，接著就可以在 template 裡面使用子元件了。

父元件 App.vue

vue
<script>
import ChildComp from './ChildComp.vue'
  
export default {
  // register child component
  components: {
    ChildComp
  }
}
</script>

<template>
  <!-- render child component -->
  <ChildComp />
</template>

子元件 ChildComp.vue

vue
<template>
  <h2>A Child Component!</h2>
</template>

props #

藉由在 template 的子元件中，加入 :msg="greeting" 將 props傳給子元件。

父元件 App.vue

vue
<script>
import ChildComp from './ChildComp.vue'

export default {
  components: {
    ChildComp
  },
  data() {
    return {
      greeting: '接收到父元件的 props 了'
    }
  },
  props: {
    msg: String
  }
}
</script>

<template>
  <ChildComp :msg="greeting"/>
</template>

子元件 ChildComp.vue

vue
<script>
export default {
  props: {
    msg: String
  }
}
</script>

<template>
  <h2>{{ msg || '還沒有 props 傳過來' }}</h2>
</template>
